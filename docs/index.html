Overview

Understanding Storage V1 architecture and concepts

What is Storage V1?

Storage V1 is a unified file storage service that abstracts away storage backend details. The same code runs on MinIO (self-hosted), AWS S3, Cloudflare R2, or Backblaze B2.

Core Concepts

Content Addressable ID (CID)

Every file is identified by its SHA-256 hash, not by filename. Same bytes → same CID always.

Tenant Partitioning

Storage key format: tenant_{ID}/{CID}. TenantID is resolved automatically from request context.

Deduplication

Upload same file twice by same tenant → stored once. Write skipped on duplicate. Caller gets same CID back instantly.

Backup Strategy

Daily restic snapshots → Backblaze B2. 12-month retention. Point-in-time restore available.

Features

What Storage V1 provides

Core Features

✓ Content-Addressable Storage

Files identified by SHA-256 hash. Same content → same ID. Enables deduplication and integrity verification.

✓ Tenant Isolation

Complete data separation per tenant. Storage key includes tenant ID. Cross-tenant access cryptographically impossible.

✓ Automatic Deduplication

Upload same file twice → stored once per tenant. Saves storage and bandwidth. Automatic, no code needed.

✓ Backend Agnostic

Same code works with MinIO, AWS S3, R2, B2. Switch backends with configuration only. No code changes.

✓ Disk Capacity Guard

Automatic check before every write. If disk ≥ 90% (configurable), writes rejected with HTTP 507. Prevents disk-full disasters.

✓ Logging Safe

Raw bytes never logged. Only CID, tenant, filename, mime-type logged. GDPR/HIPAA compliant.

✓ Built-in Backup

Daily restic snapshots to Backblaze B2. Encrypted. 12-month retention. Point-in-time restore available.

✓ Health Checks

MinIO health endpoints. Prometheus metrics. Docker health check support. Automatic alerts at 70%, 85%, 95% disk usage.

API Reference

IObjectStorageProvider interface

StoreAsync

Task<string> StoreAsync(
    Stream stream,
    string? fileName,
    string? mimeType,
    CancellationToken ct = default);

Stores a file and returns its CID (content-addressable ID).

Returns: CID (SHA-256 hash as lowercase hex string)

Throws: StorageCapacityException if disk full

GetStreamAsync

Task<Stream> GetStreamAsync(
    string cid,
    CancellationToken ct = default);

Retrieves a file stream by CID for the current tenant.

Returns: Open stream (no buffering)

Throws: FileNotFoundException if CID not found

ExistsAsync

Task<bool> ExistsAsync(
    string cid,
    CancellationToken ct = default);

Checks if a file exists for the current tenant.

Returns: true if exists, false otherwise

FAQ

Frequently asked questions

General Questions

What is a CID?

Content Addressable ID. It's the SHA-256 hash of file bytes. Same file → same CID always.

Can I access files from another tenant?

No. Storage key includes tenant ID. Cross-tenant access is impossible. System checks tenant context on every request.

What happens if I upload the same file twice?

First upload: file stored. Second upload: dedup hit, returned immediately. Same CID. Zero additional storage.

How long are backups kept?

Daily snapshots: 7. Weekly: 4. Monthly: 12. Annual: 2. Total: 12+ months via restic retention policy.

Operational Questions

What happens if disk fills up?

At 90% (default), StoreAsync throws StorageCapacityException. HTTP 507 returned to clients. Automatic safeguard.

How do I monitor disk usage?

Query minio_node_disk_used_bytes from Prometheus. Set alerts at 70%, 85%, 95%.

Can I restore a single file?

Yes. restic snapshots are per-file retrievable. Contact ops with CID + date range.

Cost Questions

Which backend is cheapest?

Backblaze B2: ~$0.006/GB/month. AWS S3: ~$0.023/GB/month. MinIO: only disk cost.

How much savings from deduplication?

20-30% on average. Payroll (templates): 30%. Audio: 10-20%. AI outputs: 30-50%.

Security Questions

Is my data encrypted?

In transit: HTTPS. At rest: depends on backend. MinIO supports encryption. B2 encrypts by default.

Are file bytes logged?

Never. Logs contain only: CID, TenantID, FileName, MimeType. No raw data. GDPR/HIPAA compliant.
